## 2. 스토리지 유형
  AWS 스토리지 서비스는 블록 스토리지, 파일 스토리지, 객체 스토리지이다.
 
  ![image](https://github.com/user-attachments/assets/99c93fb6-e170-447d-aa5c-d93d6d360e3c)

  ### 파일 스토리지
  : 블록 스토리지 디바이스에 대한 데이터 읽기 및 쓰기를 포맷하고 관리하는 운영 체제를 사용해 생성됨
    - 디렉터리 트리 계층 구조에서 데이터를 파일로 저장하는 데에 주로 사용됨
  ### 블록 스토리지
  : 하드웨어 스토리지 디바이스 또는 드라이브가 포맷되어 사용을 위해 컴퓨팅 시스템에 연결된 디스크 또는 볼륨인 원시 스토리지
  - 파일 스토리지는 파일을 단일 단위로 취급하지만 블록 스토리지는 파일을 고유한 주소를 가진 블록이라는 고정 크기의 데이터 청크로 파일을 분할함
  - 각 블록은 주소 지정이 가능하므로 블록 검색이 효율적임
  - 파일의 문자를 변경하는 경우 해당 문자가 포함된 블록만 변경하면 되기 때문에 스토리지 솔루션이빠르고 대역폭을 적게 사용함
  > 블록 스토리지는 대기 시간이 짧은 작업에 최적화되어 있으므로 데이터베이스 또는 전사적 자원 관리(ERP) 시스템과 같은 고성능 엔터프라이즈 워크로드를 위한 일반적인 스토리지 선택이며 대기 시간이 짧은 스토리지가 필요함
  ### 객체 스토리지
  : 블록 스토리지 위에 구축됨, 객체 스토리지는 블록 스토리지 디바이스에 대한 데이터 읽기 및 쓰기를 포맷하고 관리하는 운영 체제를 사용하여 생성됨
  - 미리 결정된 크기를 사용해 구성된 더 큰 블록 집합임
  - 파일 버전 관리, 파일 추적 및 파일 보존 지원함
  - 각 객체는 고유 식별자를 가진 파일로 파일에서 한 문자를 변경 시 전체 파일을 업데이트해야 함
  - 거의 모든 유형의 데이터를 저장할 수 있음
  > 대용량 데이터 집합, 미디어 자산과 같은 비정형 파일, 사진과 같은 정적 자산을 저장할 때 유용함
  ### 기존 스토리지 시스템과 다시 연결
  - 클라우드의 블록 스토리지는 직접 연결 스토리지(DAS) 또는 스토리지 영역 네트워크(SAN)과 유사함
  - 파일 스토리지 시스템은 네트워크 연결 스토리지(NAS) 서버세서 지원되는 경우가 많음

## 3. Amazon EC2 인스턴스 스토리지 및 Amazon Elastic Block Store
  ### Amazon EC2 인스턴스 스토어
  - 인스턴스 스토어는 인스턴스에 임시 블록 수준 스토리지를 제공함
  - 호스트 컴퓨터에 물리적으로 연결된 디스크에 위치함
  - 인스턴스의 수명 주기와 연동됨
  > 버퍼, 캐시 및 임시 콘텐츠와 같이 자주 변경되는 정보를 임시 저장하는 데에 사용됨<br>
  > 웹 서버의 로드밸런싱된 풀과 같은 인스턴스 집합 전체에 복제되는 데이터

  - 인스턴스 스토어 수명 (다음과 같은 상황에 데이터 손실됨)
      - 기본 디스크 드라이브가 실패함
      - 인스턴스가 중지됨
      - 인스턴스가 최대 절전 모드로 전환됨
      - 인스턴스가 종료됨
    
  ### Amazon EBS
  - AS 블록 스토리지 포트폴리오는 인스턴스 스토리지와 Amazon EBS라는 두 가지 유형의 블록 스토리지 서비스로 구성됨
  - EC2 인스턴스에 연결 가능한 블록 수준 스토리지 디바이스임
  - 대부분의 EBS 볼륨은 한 번으 한 대의 컴퓨터(인스턴스)와만 연결 가능함
  - 한 EC2 인스턴스에서 EVS 볼륨을 분리하고 동일 가용 영역의 다른 EC2인스턴스에 연결해 해당 인스턴스 데이터에 엑세스함
  - EBS 볼륨 크기 조정
    - 최대 크기 제한을 초과하지 않다면 볼륨 크기를 늘림(최대 16TB)
    - 여러 볼륨을 단일 EC2 인스턴스에 연결함(EC2:EBS 볼륨 = 1:N)
    ![image](https://github.com/user-attachments/assets/e2ebc950-225f-4a56-a54c-b25f792145f2)
  - 장점
    - 고가용성: EBS 볼륨을 생성하면 단일 장애 지점으로 인한 데이터 손실을 방지하기 위해 볼륨이 가용 영역에 자동으로 복제됨
    - 데이터 지속성: 인스턴스가 없는 경우에도 스토리지가 유지됨
    - 데이터 암호화: 모든 EBS 볼륨은 암호화를 지원함
    - 유연성: EBS 볼륨은 즉시 변경을 지원함. 인스턴스를 중지하지 않고도 볼륨 유형, 볼륨 크기 및 IOPS(초당 입출력 작업 수) 용량을 수정할 수 있음
    - 백업: Amazon EBS는 모든 EBS 볼륨의 백업을 생성할 수 있는 기능을 제공함
